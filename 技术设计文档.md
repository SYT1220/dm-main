# 技术设计文档

## 1. 引言

### 1.1 编写目的

本文档详细描述宿舍管理系统的技术架构、数据库设计、接口设计等，为开发人员提供技术指导，同时为后续系统维护提供参考。

### 1.2 适用范围

本文档适用于系统设计人员、开发人员、测试人员以及系统维护人员。

## 2. 系统架构

### 2.1 总体架构

系统采用经典的分层架构设计，自顶向下分为表现层、业务逻辑层、数据访问层和数据层。

表现层负责与用户交互，展示数据和接收用户输入，基于 Bootstrap 和 jQuery 开发，实现响应式页面设计。

业务逻辑层负责业务逻辑处理、权限控制、数据校验等，基于 Spring Boot 框架开发。

数据访问层负责数据持久化操作，基于 Spring Data JPA 实现，简化数据库操作。

数据层采用 MySQL 8.0 关系型数据库，存储系统所有业务数据。

### 2.2 技术栈

后端框架：Spring Boot 2.6.1，提供快速开发能力和丰富的生态系统。

持久层框架：Spring Data JPA，简化数据库操作，提高开发效率。

数据库：MySQL 8.0，成熟稳定的关系型数据库。

前端框架：Bootstrap 4.x，实现响应式页面设计。

前端交互库：jQuery 3.6.0，简化 DOM 操作和 AJAX 请求。

外部服务：百度 AI 人脸识别服务、阿里云 AI 助手服务。

## 3. 数据库设计

### 3.1 数据库概念设计

系统包含六个核心实体：管理员、楼宇、宿舍、辅导员、学生、住宿记录。

管理员实体：存储系统管理员信息，包括账号、密码、姓名等。

楼宇实体：存储宿舍楼宇信息，包括名称、位置等，与辅导员为多对多关系。

宿舍实体：存储宿舍房间信息，包括编号、楼层、床位数量等，属于某个楼宇。

辅导员实体：存储辅导员信息，包括工号、姓名、性别等，与楼宇为多对多关系。

学生实体：存储学生信息，包括学号、姓名、性别等。

住宿记录实体：存储学生住宿信息，包括学生、宿舍、入住时间等。

### 3.2 数据库表结构

#### 3.2.1 管理员表（admin）

字段包括：ID（主键）、创建时间、更新时间、有效标志、版本号、姓名（唯一）、密码。

#### 3.2.2 楼宇表（building）

字段包括：ID（主键）、创建时间、更新时间、有效标志、版本号、名称（唯一）、位置。

#### 3.2.3 楼宇辅导员关联表（building_dormitory_manager）

字段包括：ID（主键）、创建时间、更新时间、有效标志、版本号、楼宇 ID（外键）、辅导员 ID（外键），楼宇 ID 和辅导员 ID 组合唯一。

#### 3.2.4 宿舍表（dormitory）

字段包括：ID（主键）、创建时间、更新时间、有效标志、版本号、编号（唯一）、楼层、最大人数、已住人数、楼宇 ID（外键）。

#### 3.2.5 辅导员表（dormitory_manager）

字段包括：ID（主键）、创建时间、更新时间、有效标志、版本号、工号（唯一）、姓名、密码、性别。

#### 3.2.6 学生表（student）

字段包括：ID（主键）、创建时间、更新时间、有效标志、版本号、学号（唯一）、姓名、密码、性别。

#### 3.2.7 住宿表（live）

字段包括：ID（主键）、创建时间、更新时间、有效标志、版本号、学生 ID（外键）、宿舍 ID（外键）、入住时间。

### 3.3 数据库关系设计

楼宇与辅导员为多对多关系，通过 building_dormitory_manager 关联表实现。

楼宇与宿舍为一对多关系，一个楼宇包含多间宿舍。

宿舍与住宿为一对多关系，一间宿舍可有多条住宿记录（历史）。

学生与住宿为一对多关系，一个学生可有多条住宿记录（历史）。

## 4. 接口设计

### 4.1 接口规范

所有接口采用 RESTful 风格设计，使用 HTTP 协议。请求和响应数据格式为 JSON，字符编码为 UTF-8。

### 4.2 登录接口

接口路径：/login，请求方法：POST，请求参数：用户名、密码，响应数据：登录结果、用户信息。

接口路径：/face/detect，请求方法：POST，请求参数：人脸图像 Base64 编码，响应数据：识别结果、用户信息。

### 4.3 楼宇管理接口

接口路径：/building/list，请求方法：GET，请求参数：搜索条件，响应数据：楼宇列表。

接口路径：/building/add，请求方法：POST，请求参数：楼宇信息，响应数据：操作结果。

接口路径：/building/edit，请求方法：POST，请求参数：楼宇信息，响应数据：操作结果。

接口路径：/building/delete，请求方法：POST，请求参数：楼宇 ID，响应数据：操作结果。

### 4.4 宿舍管理接口

接口路径：/dormitory/list，请求方法：GET，请求参数：搜索条件，响应数据：宿舍列表。

接口路径：/dormitory/add，请求方法：POST，请求参数：宿舍信息，响应数据：操作结果。

接口路径：/dormitory/edit，请求方法：POST，请求参数：宿舍信息，响应数据：操作结果。

接口路径：/dormitory/delete，请求方法：POST，请求参数：宿舍 ID，响应数据：操作结果。

### 4.5 学生管理接口

接口路径：/student/list，请求方法：GET，请求参数：搜索条件，响应数据：学生列表。

接口路径：/student/add，请求方法：POST，请求参数：学生信息，响应数据：操作结果。

接口路径：/student/import，请求方法：POST，请求参数：Excel 文件，响应数据：导入结果。

接口路径：/student/edit，请求方法：POST，请求参数：学生信息，响应数据：操作结果。

接口路径：/student/delete，请求方法：POST，请求参数：学生 ID，响应数据：操作结果。

### 4.6 辅导员管理接口

接口路径：/dormitoryManager/list，请求方法：GET，请求参数：搜索条件，响应数据：辅导员列表。

接口路径：/dormitoryManager/add，请求方法：POST，请求参数：辅导员信息，响应数据：操作结果。

接口路径：/dormitoryManager/edit，请求方法：POST，请求参数：辅导员信息，响应数据：操作结果。

接口路径：/dormitoryManager/delete，请求方法：POST，请求参数：辅导员 ID，响应数据：操作结果。

### 4.7 住宿管理接口

接口路径：/live/checkin，请求方法：POST，请求参数：学生 ID、宿舍 ID，响应数据：操作结果。

接口路径：/live/checkout，请求方法：POST，请求参数：学生 ID，响应数据：操作结果。

接口路径：/live/transfer，请求方法：POST，请求参数：学生 ID、新宿舍 ID，响应数据：操作结果。

### 4.8 外部服务接口

百度人脸识别接口：调用百度 AI 开放平台的人脸检测和识别接口，实现人脸注册和人脸登录功能。

阿里云 AI 助手接口：调用阿里云的 AI 对话接口，实现智能问答功能。

## 5. 模块设计

### 5.1 用户管理模块

模块包含登录、注册、密码管理、人脸注册等功能。登录功能验证用户身份，注册功能添加新用户，密码管理功能修改用户密码，人脸注册功能采集和保存人脸特征。

### 5.2 楼宇管理模块

模块包含楼宇的增删改查功能和辅导员分配功能。使用 BuildingEntity 实体类映射楼宇表，使用 BuildingRepository 接口进行数据访问，使用 BuildingController 处理 HTTP 请求。

### 5.3 宿舍管理模块

模块包含宿舍的增删改查功能和入住统计功能。使用 DormitoryEntity 实体类映射宿舍表，使用 DormitoryRepository 接口进行数据访问，使用 DormitoryController 处理 HTTP 请求。

### 5.4 学生管理模块

模块包含学生的增删改查功能和批量导入功能。使用 StudentEntity 实体类映射学生表，使用 StudentRepository 接口进行数据访问，使用 StudentController 处理 HTTP 请求。

### 5.5 辅导员管理模块

模块包含辅导员的增删改查功能和楼宇分配功能。使用 DormitoryManagerEntity 实体类映射辅导员表，使用 DormitoryManagerRepository 接口进行数据访问，使用 DormitoryManagerController 处理 HTTP 请求。

### 5.6 住宿管理模块

模块包含入住、退宿、调宿等功能。使用 LiveEntity 实体类映射住宿表，使用 LiveRepository 接口进行数据访问，使用 LiveController 处理 HTTP 请求。

### 5.7 人脸识别模块

模块包含人脸注册和人脸检测功能。使用百度 AI SDK 调用人脸识别服务，使用 FaceRecognitionService 接口定义服务方法，使用 FaceRecognitionServiceImpl 实现服务逻辑。

## 6. 安全设计

### 6.1 认证机制

系统实现用户认证机制，未登录用户不能访问系统功能。使用 Session 管理用户登录状态，设置合理的超时时间。

### 6.2 权限控制

系统实现基于角色的权限控制，不同角色只能访问授权的功能。使用拦截器检查用户权限，拦截未授权的访问。

### 6.3 密码加密

用户密码采用加密存储，防止数据泄露。使用 MD5 或其他加密算法对密码进行加密。

### 6.4 日志记录

系统记录关键操作日志，包括操作人、操作时间、操作内容等。日志存储在数据库和日志文件中，便于追溯。

## 7. 性能优化

### 7.1 数据库优化

使用连接池管理数据库连接，提高连接复用率。优化 SQL 查询语句，避免全表扫描。为常用查询字段建立索引，提高查询速度。

### 7.2 缓存优化

对于频繁查询且不常变化的数据，使用缓存减少数据库访问。使用内存缓存存储配置信息等。

### 7.3 前端优化

使用 CDN 加速静态资源加载。启用 Gzip 压缩减少传输数据量。优化图片大小，使用合适的格式。

## 8. 部署设计

### 8.1 服务器配置

应用服务器：配置 JDK 1.8+，内存≥4GB，硬盘≥50GB。数据库服务器：配置 MySQL 8.0+，内存≥4GB，硬盘≥50GB。

### 8.2 网络配置

开放 80 端口用于 HTTP 访问，开放 443 端口用于 HTTPS 访问（可选），开放 8080 端口用于应用服务，开放 3306 端口用于数据库连接。

### 8.3 备份策略

数据库每天进行增量备份，每周进行全量备份。备份文件存储在独立的存储设备上，定期测试数据恢复流程。
